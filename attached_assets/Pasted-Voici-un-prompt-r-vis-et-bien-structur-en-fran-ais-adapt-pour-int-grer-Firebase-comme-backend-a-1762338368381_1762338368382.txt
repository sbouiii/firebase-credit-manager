Voici un prompt révisé et bien structuré en français, adapté pour intégrer Firebase comme backend (au lieu de Node.js/Express/MongoDB). Firebase gérera l'authentification, la base de données (via Firestore), les notifications (via Firebase Cloud Messaging ou Functions), et d'autres aspects. Ce prompt est conçu pour guider une IA générative à générer du code et des idées complètes. Vous pouvez le copier-coller directement.

Prompt pour IA :
"Tu es un expert en développement web full-stack, spécialisé dans les applications de gestion d'entreprise avec un focus sur l'UX/UI moderne et innovant. Développe une application web complète pour la gestion de crédits dans une boutique (par exemple, gestion des crédits clients, suivis des paiements, historiques, rappels, etc.). L'application doit être construite avec les technologies suivantes : React.js pour le frontend, Firebase pour le backend (incluant Firestore pour la base de données NoSQL, Firebase Authentication pour la sécurité, Firebase Cloud Functions pour les logiques serveur si nécessaires, et Firebase Cloud Messaging pour les notifications), et Tailwind CSS pour le styling afin d'assurer un design responsive et moderne.
Fonctionnalités principales requises :

Authentification sécurisée avec Firebase Auth (login/register via email/mot de passe, Google, etc., avec rôles admin/vendeur/client gérés via custom claims).
Tableau de bord principal avec vue d'ensemble : total des crédits en cours, crédits payés, crédits en retard, graphiques analytiques (utilise Chart.js pour des visualisations dynamiques, en fetching les données de Firestore).
Gestion des clients : ajout/modification/suppression de clients via Firestore, avec profils incluant nom, contact, historique de crédits.
Gestion des crédits : création de nouveaux crédits (montant, date d'échéance, taux d'intérêt si applicable), mise à jour des paiements partiels, calcul automatique des soldes restants (via queries Firestore ou Cloud Functions pour les calculs complexes).
Système de rappels : notifications automatiques par email (via Firebase Extensions ou Cloud Functions avec Nodemailer) ou push notifications (via Firebase Cloud Messaging) pour les crédits en retard.
Rapports et exports : génération de rapports PDF/CSV sur les crédits (utilise des libs comme pdfmake ou csv-writer, déclenchées via Cloud Functions pour l'export server-side).
Recherche et filtres avancés : recherche par client, date, statut de crédit via queries Firestore.

Design et UX/UI modernes :

Utilise un thème sombre/clair switchable avec des couleurs vives et minimalistes (inspiré de designs comme ceux de Material UI ou Shadcn UI).
Interface responsive pour mobile/desktop, avec navigation fluide via sidebar ou navbar.
Animations innovantes et attirantes :

Transitions fluides avec Framer Motion (ex. : fade-in pour les modals, slide pour les menus déroulants).
Événements interactifs : hover effects avec scaling et glow sur les cartes de crédits, drag-and-drop pour réorganiser les listes de clients.
Animations engageantes : confetti ou particules sur validation de paiement réussi (utilise particles.js), progress bars animées pour les soldes de crédits.
Événements innovants : tooltips animés avec Lottie animations pour expliquer les fonctionnalités, ou un chatbot intégré (via une lib comme React Chatbot Kit) pour assister les utilisateurs en temps réel.



Étapes de développement :

Fournis une structure de projet complète (arborescence des fichiers, incluant la configuration Firebase).
Génère le code frontend complet (composants React, pages, hooks pour interagir avec Firebase via le SDK Firebase).
Explique la configuration Firebase : initialisation du projet Firebase, setup de Firestore (règles de sécurité), Auth, et Cloud Functions si needed.
Inclu des instructions pour l'installation (npm install firebase, etc.), le déploiement (sur Firebase Hosting pour le frontend, et Functions pour le backend), et les tests (avec Jest).
Assure la sécurité : règles Firestore pour empêcher les accès non autorisés, validation des inputs côté client et serveur.
Optimise pour les performances : utilisation de queries indexées dans Firestore, lazy loading, caching local avec React hooks.

Rends le code modulaire, commenté et prêt à l'emploi. Si des dépendances sont nécessaires, liste-les avec des commandes npm install (ex. : npm install firebase @firebase/firestore framer-motion chart.js etc.). Commence par un overview du projet avant de plonger dans le code."
